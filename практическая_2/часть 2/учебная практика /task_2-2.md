# Задание 2.2. Проектирование ролей

## Матрица привилегий

| Объект / Операция | guest | employee | manager | admin | superuser |
|---|---|---|---|---|---|
| `app.users` - SELECT | - | Свой профиль | Пользователи своих проектов | Все | Все |
| `app.users` - INSERT/UPDATE | - | - | - | ✓ | ✓ |
| `app.projects` - SELECT | - | Проекты, в которых участвует | Свои проекты | Все | Все |
| `app.projects` - INSERT | - | - | ✓ | ✓ | ✓ |
| `app.projects` - UPDATE | - | - | Свои проекты | Все | Все |
| `app.projects` - DELETE | - | - | Свои проекты | Все | Все |
| `app.tasks` - SELECT | - | Свои задачи и задачи своих проектов | Все задачи своих проектов | Все | Все |
| `app.tasks` - INSERT | - | В доступных проектах | В своих проектах | Все | Все |
| `app.tasks` - UPDATE | - | Свои задачи | Все задачи своих проектов | Все | Все |
| `app.tasks` - DELETE | - | - | Все задачи своих проектов | Все | Все |
| `app.comments` - SELECT | - | Комментарии к доступным задачам | Комментарии своих проектов | Все | Все |
| `app.comments` - INSERT | - | ✓ | ✓ | ✓ | ✓ |
| `app.task_history` - SELECT | - | История своих задач | История задач своих проектов | Все | Все |
| `app.access_logs` - SELECT | - | - | - | ✓ | ✓ |
| DDL операции (`CREATE/ALTER/DROP`) | - | - | - | В своей схеме | Все |

## Краткое обоснование

- `guest` не работает с внутренними данными системы управления задачами, поэтому прав почти не имеет.
- `employee` работает только со своими задачами, своими комментариями и видит только те проекты, где участвует.
- `manager` управляет своими проектами и всеми задачами внутри этих проектов.
- `admin` управляет всеми прикладными объектами системы и может выполнять DDL-операции в своей схеме.
- `superuser` имеет полный доступ ко всей базе данных и всем объектам.
